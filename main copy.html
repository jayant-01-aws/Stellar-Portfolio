The WoodLandv4 project follows a clean inheritance chain built on top of Unreal Engine's APawn base class.

  • PrimaryActorTick
  • SetupPlayerInputComponent
  • GetController
→

AVPlane

  • PlaneBody (UStaticMesh)
  • R_Muzzel (USceneComp)
  • L_Muzzel (USceneComp)
→

  • SpringArm
  • Camera
  • Replicated properties
  • Server RPCs

VPlane.h / VPlane.cpp — Base Aircraft Pawn

▼

AVPlane is the base pawn class that defines the physical structure of every aircraft in the game. It attaches a UStaticMeshComponent as the fuselage and two USceneComponents as left and right gun muzzle mount points.

// VPlane.h — Declares base aircraft structure class WOODLANDV4_API AVPlane : public APawn { GENERATED_BODY() protected: UPROPERTY(VisibleAnywhere, Category = "Mesh") UStaticMeshComponent* PlaneBody; // main fuselage UPROPERTY(VisibleAnywhere, Category = "Mesh") USceneComponent* R_Muzzel; // right gun mount USceneComponent* L_Muzzel; // left gun mount };

Key design note: PlaneBody is attached to RootComponent. Both muzzle points attach to PlaneBody, inheriting its world transform. This makes rotation and world-position look-ups trivial for bullet spawn logic.

JellyPlane.h / JellyPlane.cpp — Combat Aircraft

▼

AJellyPlane extends AVPlane with full combat capabilities: camera rigging, throttle system, replicated health, server RPCs for all inputs, and a special area-of-effect ability.

// Server-authoritative fire logic void AJellyPlane::serverFire_Implementation() { FVector Location; FRotator Rotation; GetController()->GetPlayerViewPoint(Location, Rotation); FVector End = Location + Rotation.Vector() * 10000.f; FHitResult Hit; bool bSuccess = GetWorld()->LineTraceSingleByChannel( Hit, Location, End, ECC_GameTraceChannel1); if (bSuccess) { FPointDamageEvent DamageEvent(10.f, Hit, -Rotation.Vector(), nullptr); if (AActor* HitActor = Hit.GetActor()) HitActor->TakeDamage(10.f, DamageEvent, GetController(), this); } }
// TakeDamage override with health clamping float AJellyPlane::TakeDamage(float DamageAmount, ...) { float Applied = Super::TakeDamage(DamageAmount, ...); CurrentHealth = FMath::Clamp(CurrentHealth - DamageAmount, 0.f, 500.f); if (CurrentHealth <= 0) { OnDeath(DamageCausor); Destroy(); } return Applied; }

Replication: CurrentHealth, ThrottleAmount, Range, bThrottleUp, bThrottleDown, and Flash are all replicated using DOREPLIFETIME macros. All input goes through Server RPCs (serverFire, serverTurn, serverLookUp etc.) ensuring cheating prevention in multiplayer.

// Throttle physics — called every 0.05s by timer void AJellyPlane::CalculateSpeed() { float Speeed = FMath::Clamp(ThrottleAmount * Speed, 0.f, Speed * 10); FVector ForwardVector = PlaneBody->GetForwardVector() * Speeed; FVector PVelocity = PlaneBody->GetPhysicsLinearVelocity(); PlaneBody->SetPhysicsLinearVelocity( FMath::Lerp(PVelocity, ForwardVector, 0.01f)); }

CrosshairHUD.h / CrosshairHUD.cpp — On-Screen HUD

▼

ACrosshairHUD extends AHUD and renders a UTexture2D crosshair centered on screen. The DragCrossHair method shifts the crosshair by an offset — used to visualise look input momentum.

// Centered crosshair draw void ACrosshairHUD::DrawHUD() { Super::DrawHUD(); const FVector2D Center(Canvas->ClipX * 0.5f, Canvas->ClipY * 0.5f); const FVector2D DrawPos(Center.X - 5.f, Center.Y - 5.f); FCanvasTileItem TileItem(DrawPos, CrosshairTex->Resource, FLinearColor::White); TileItem.BlendMode = SE_BLEND_Translucent; Canvas->DrawItem(TileItem); } // Dynamic crosshair with offset (for aim-sway) void ACrosshairHUD::DragCrossHair(float value, float value1) { const FVector2D DrawPos( Center.X - 5.f + value, Center.Y - 5.f + value1); // ... same draw logic with offset applied }

The texture is loaded at construction via ConstructorHelpers::FObjectFinder pointing to /Game/Crosshair/FirstPersonCrosshair. The GameMode registers ACrosshairHUD as the HUDClass so UE5 automatically spawns it for every local player.
